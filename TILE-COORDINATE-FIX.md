# Tile Coordinate System Fix

**Date:** October 4, 2025  
**Issue:** Map tiles not displaying - requesting wrong tile coordinates  
**Root Cause:** INCORRECT RESOLUTIONS in tile grid configuration

---

## The Problem (Root Cause)

### Current Resolution Generation (BROKEN)

```csharp
private double[] GenerateResolutions(int levels)  // levels = 9
{
    var resolutions = new double[levels];  // Only 9 elements!
    for (var i = 0; i < levels; i++)
    {
        resolutions[i] = Math.Pow(2, levels - i - 1);  // Wrong formula!
    }
    return resolutions;
}
```

**Result:**
- Generates: `[256, 128, 64, 32, 16, 8, 4, 2, 1]` (9 elements)
- Should be: `[512, 256, 128, 64, 32, 16, 8, 4, 2, 1]` (10 elements)

### Why This Breaks Everything

**With wrong resolutions (128 at zoom 8):**
- OpenLayers thinks each tile covers: `128 blocks/pixel × 256 pixels = 32,768 blocks`
- Actual tiles cover: `512 blocks` (at zoom 8)
- **Mismatch by 64x!**

**Result:**
- OpenLayers calculates wrong tile coordinates
- Requests tiles that don't exist
- Blank map ❌

---

## The Fix

### Corrected Resolution Generation

```csharp
private double[] GenerateResolutions(int maxZoom)  // maxZoom = 9
{
    // Need maxZoom + 1 elements for zooms 0 through maxZoom
    var resolutions = new double[maxZoom + 1];
    for (var i = 0; i <= maxZoom; i++)
    {
        // Resolution = blocks per pixel at this zoom level
        // Zoom 0: 2^9 = 512 blocks/pixel (far out)
        // Zoom 9: 2^0 = 1 block/pixel (zoomed in)
        resolutions[i] = Math.Pow(2, maxZoom - i);
    }
    return resolutions;
}
```

**Result:**
- Zoom 0: 512 blocks/pixel → tiles cover 512 × 256 = 131,072 blocks ✅
- Zoom 8: 2 blocks/pixel → tiles cover 2 × 256 = 512 blocks ✅
- Zoom 9: 1 block/pixel → tiles cover 1 × 256 = 256 blocks ✅

## Implementation

### Backend Changes

**File:** `VintageAtlas/Web/API/MapConfigController.cs`

```csharp
private double[] GenerateResolutions(int maxZoom)
{
    // Generate resolutions for zooms 0 through maxZoom (inclusive)
    var resolutions = new double[maxZoom + 1];
    for (var i = 0; i <= maxZoom; i++)
    {
        // Resolution = blocks per pixel at this zoom level
        // At zoom 0 (far out): high resolution number (many blocks per pixel)
        // At maxZoom (zoomed in): resolution = 1 (one block per pixel)
        resolutions[i] = Math.Pow(2, maxZoom - i);
    }
    return resolutions;
}
```

**Update call site:**
```csharp
// Calculate resolutions for OpenLayers
// BaseZoomLevel = 9 means zooms 0-9 (10 levels total)
var tileResolutions = GenerateResolutions(_config.BaseZoomLevel);
var viewResolutions = GenerateResolutions(_config.BaseZoomLevel + 3); // Extra for smooth viewing
```

### No Frontend Changes Needed!

The frontend already handles the resolutions correctly - it just needs the correct values from the backend.

## Additional Improvements (Optional)

### 1. Use Absolute Coordinates (Simpler)

For even cleaner implementation, switch to absolute coordinates:

```csharp
// In MapConfigController.cs
if (_config.AbsolutePositions)
{
```

**Benefits:**

- Simpler debugging

**Trade-off:**
- Coordinates displayed as large numbers (e.g., 512000)
